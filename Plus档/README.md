# Plus 档 · 叠加说明

**高上限 · 需人工审核 · 长主线 + 多支线**

---

## 前提

**先配好[基础档](../01-基础档-Standard/README.md)。** Plus 档是在基础档之上叠加，不是替代。

Plus 档的成本是**人工审核**：插件会自动从对话里抓记忆条目，但抓完不会直接生效——先进「不启用」的仓库世界书，由你审核后再分发到对应世界书。愿意持续做这件事，叙事上限极高；不愿意，就留在基础档。

---

## 本档包含的文件

| 文件 | 是什么 | 装到哪 |
|---|---|---|
| `认知边界书.json` | 认知边界书，记录「谁知道什么」——披露、共同经历、认知边界 | 世界书（绑为**附加知识书**） |
| `窗口世界书_模板.json` | 窗口世界书，剧情线的状态锚 / 阶段基调 / 计划账本，全为脱敏模板 | 聊天世界书 |
| `STMB与WREC工作流.md` | 双插件工作流说明 | 见文件内 |

---

## 叠加步骤

**1. 导入认知边界书**
世界书页 → 导入 `认知边界书.json` → 在角色卡的「附加知识书」里绑定它。
> 里面两条「已知/条目模板」「能力/条目模板」是**禁用状态**，别删——它们是 STMB 自动抓取的落点格式参照。

**2. 导入窗口世界书**
导入 `窗口世界书_模板.json` 作为当前聊天的世界书。
> 全部为脱敏模板（原剧情线内容已移除）。照模板结构填你自己的剧情线；带 `{{user}}` 占位符的照旧保留。

**3. 装插件并配提示词**
按 `STMB与WREC工作流.md` 走：装 STMB、WREC，粘贴各自提示词，把两个插件的模型温度降到 0.3。

**4. （可选）导入 Regex**
用 TTS 输出才需要。扩展 → Regex → 导入 `regex-tts标签.json`。

---

## 维护工作流一句话版

```
日常 RP → STMB 抓「计划/能力/认知」条目进仓库世界书
        → 人工审核 → 分发到对应世界书
        → WREC 扫上下文刷新条目状态字段 → 继续 RP
        （全程开着基础档的滚动总结当保险）
```

详见 `STMB与WREC工作流.md`。
