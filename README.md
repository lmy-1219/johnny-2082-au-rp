## 这个仓库分两档

按**你愿意投入多少维护成本**分档，不是按功能分。先想清楚要玩多长、愿不愿意手动审核记忆条目，再决定下哪一档。

### 📦 基础档（Standard）— 开箱即用，零维护

装好就能玩。角色卡自带内嵌世界书，配上全局世界书、模型预设、滚动总结即可。

- **适用**：200–400 轮以内的日常向叙事
- **表现**：剧情连贯、角色不跑形、零维护成本
- **实测**：单轮消息约 1500 tokens
- **边界（重要）**：**不支持**这 200–400 轮内跨度过大的剧情时间线、或塞进太多里程碑事件。只支持日常向。要长主线 / 多支线 / 大时间跨度，请上 Plus 档。

→ 进 [`01-基础档-Standard/`](./基础档/README.md)

### 🔧 Plus 档 — 高上限，需人工审核

在基础档之上叠加认知边界书、窗口世界书、双插件工作流。插件自动抓取记忆条目，你人工审核后分发。

- **适用**：长主线 + 多支线的复杂叙事
- **成本**：需要定期审核插件抓取的条目内容
- **回报**：只要坚持维护，叙事上限极高

→ 进 [`02-Plus档/`](./Plus档/README.md)（**需先配好基础档**）

## 文件对应关系总表

| 文件 | 作用 | 归档 |
|---|---|---|
| `Johnny_Silverhand_v2.json` | **角色卡主体**，已内嵌角色世界书 15 条（人格反应／关系／前史／防火墙），无需另装角色书 | 基础档 |
| `CP2082_Daily_Optimized_v3_20entry_worldinfo.json` | **全局世界书**，静态事实——夜之城运作的各个系统（势力、帮派、义体维护、黑市、中间人档案、场所地图等） | 基础档 |
| `模型预设_Claude.json` | Claude 后端的采样参数 + 提示词（含针对该模型的方向性校正） | 基础档 |
| `模型预设_Gemini.json` | Gemini 后端的采样参数 + 提示词 | 基础档 |
| `滚动总结提示词.md` | ST 自带「总结」功能的配置 + 提示词，剧情连续性保险 | 基础档 |
| `认知边界书.json` | **认知边界书**，记录「谁知道什么」——一方向另一方披露过、双方共同经历、以及明确的认知边界（油田／骨灰龛／酒店等） | Plus 档 |
| `窗口世界书_模板.json` | **窗口世界书**，剧情线运行时的状态锚 / 阶段基调 / 计划账本，全部为脱敏模板 | Plus 档 |
| `STMB与WREC工作流.md` | 双插件工作流说明——记忆抓取 + 状态刷新 + 条目改造 | Plus 档 |
| `regex-tts标签.json` | Regex 脚本，出站剥离 TTS 标签（用 TTS 才需要，可选） | Plus 档 |
